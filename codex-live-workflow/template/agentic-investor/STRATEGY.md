## 1. 市场状态引擎（Market State）

基于 NVDA 和 NASDAQ 的趋势判断，自动切换仓位：

| 状态 | 条件 | 股票仓位 | 期权仓位 | 现金 |
|------|------|---------|---------|------|
| **BULL** | NVDA > MA50 AND NASDAQ > MA50 | 70% | 20% | 10% |
| **SIDEWAYS** | NVDA 在 MA50~MA200 之间 | 60% | 10% | 30% |
| **BEAR** | NVDA < MA200 | 40% | 0% | 60% |

**MA 计算方法**：
- MA50：过去50个交易日收盘价简单移动平均
- MA200：过去200个交易日收盘价简单移动平均
- 数据来源：moomoo API `get_historical_klines`（K_QL_DAILY）

## 3. 期权策略引擎（3种策略，覆盖不同场景）

### 3.1 Covered Call — 核心收益增强（针对 GOOG、MSFT）

| 参数 | 值 |
|------|-----|
| 适用标的 | 核心仓中持仓 >$10k 的标的 |
| Strike | OTM 5-10%（现价 × 1.07） |
| DTE | 30-45 天 |
| 触发条件 | 持有正股 ≥ 100 股 AND 隐含波动率 IV > 20 |
| 退出规则 | 到期自动结算 OR 权利金已赚取 >80% → 提前平仓 |
| 年化目标 | 额外 8-15% 收益 |

### 3.2 Cash-Secured Put — 低价建仓（针对 PANW、CRWD）

| 参数 | 值 |
|------|-----|
| 适用标的 | 卫星仓中有建仓意愿的标的 |
| Strike | OTM 10-15%（现价 × 0.85） |
| DTE | 30 天 |
| 触发条件 | 目标股票不在持仓中 OR 想加仓 |
| 结果A（未行权） | 白赚权利金，继续卖 |
| 结果B（被行权） | 以折扣价买入正股 → 切换为 Covered Call |

### 3.3 Bull Call Spread — 杠杆做多（针对 NVDA、AMZN）

| 参数 | 值 |
|------|-----|
| 适用标的 | 高确信度且有明确催化剂（财报/新品） |
| 结构 | 买入 ATM Call + 卖出 OTM Call（价差 $30-50） |
| DTE | 60-90 天 |
| 最大仓位 | 总账户 5% |
| 止损 | 亏损 50% → 平仓 |
| 止盈 | 盈利 100% → 平 50%；盈利 200% → 全部平仓 |

## 4. 风控引擎

### 4.1 个股止损

| 仓位类型 | 止损线 | 操作 |
|---------|--------|------|
| 核心仓 | -20% | 减半仓 |
| 卫星仓 | -25% | 清仓 |

### 4.2 期权止损

| 策略 | 止损线 | 操作 |
|------|--------|------|
| 买方（Bull Spread） | -50% | 平仓 |
| 卖方（CC/CSP） | 亏损 > 收取权利金 2x | 平仓或 Roll 到下一月 |

### 4.3 组合级风控

| 规则 | 阈值 | 操作 |
|------|------|------|
| 单日亏损 | -3% | 停止当日所有新开仓 |
| 最大回撤 | -20% | 所有仓位减半，只保留现金 |
| 期权连亏 | 3次连续亏损 | 暂停期权交易 30 天 |
| 单票上限 | 20% | 禁止继续加仓 |
| 现金底线 | 10% | 永远保留至少 10% 现金 |

## 5. 执行流程

### 每日流程（交易日，北京时间）

| 时间 | 步骤 |
|------|------|
| **10:15** | 0. 中国白天盘后复盘：基于昨夜行情、新闻、社媒舆情、财报、宏观、基金经理/议员披露，输出 Top 3 股票 Pitch；用户决定是否推进明日 conditional paper playbook |
| **21:25** | 1. 获取 MA50/MA200 → 计算市场状态 |
| | 2. 检查持仓盈亏 → 触发止损/止盈 |
| | 3. 扫描期权到期日 → 处理即将到期的期权 |
| **21:30** | 4. 盘前信号检查 → 有信号则挂单 |
| **23:45** | 5. 睡前校准：复核用户已批准的 Top 3 pitch 候选，确认是否写入/保留/取消明日 conditional paper playbook |
| **00:00-03:45** | 6. 盘中每 15 分钟复查 → 检查 paper scenario、市场异动、新闻/舆情报警 |
| **04:00** | 7. 盘后总结 → 更新持仓记录 → 推送日报 |

### 每周流程（周六北京时间）

| 步骤 |
|------|
| 1. 计算所有股票 30 日收益率 |
| 2. 按动量排序，检查是否有 TOP/BOTTOM 3 变化 |
| 3. 检查各仓位权重偏离是否 > 5% |
| 4. 生成调仓建议（需用户确认后才执行实盘） |
| 5. 推送周报 |

### 每月流程（每月1日）

| 步骤 |
|------|
| 1. 注入新月定投资金（~$3,000） |
| 2. 全面调仓 |
| 3. 更新 Leader/Follower 股票名单 |
| 4. 生成月报 |
